微信iLink接口开发文档

本文档旨在为后端开发者提供一份完整、专业且可操作的微信iLink Bot API接入指南。内容涵盖功能概述、认证流程、核心接口说明、典型业务调用流程及Python实现示例，适用于构建基于个人微信的自动化服务或AI Agent系统。

1. 微信iLink接口功能概述

本节目标

了解iLink协议的核心定位、通信模型与能力边界，明确其在开发者生态中的角色。

微信iLink（智联）是腾讯官方推出的个人微信Bot开放通道，通过HTTP/JSON协议暴露消息收发能力，允许开发者以Bot身份与用户进行双向通信。该接口由ClawBot插件驱动，运行于用户的微信客户端中，作为消息代理将指令转发至开发者服务器。

协议定位

  • 官方性质：由腾讯微信团队发布并维护，非逆向工程产物。
  • 接入方式：无需SDK，支持标准HTTP请求直接调用。
  • 服务边界：仅作消息通道，不存储用户内容；AI逻辑和数据处理由开发者自建系统完成。

核心能力

能力 说明
消息收发 支持文本、图片、语音、文件、视频等类型
上下文绑定 通过context_token确保回复关联正确对话窗口
媒体传输 所有媒体经AES-128-ECB加密后上传至腾讯CDN
输入态控制 可发送“正在输入…”状态提示

通信模型

采用长轮询（Long Polling）机制替代WebSocket：

  • 客户端周期性发起POST请求至/ilink/bot/getupdates
  • 服务器最多挂起连接35秒，直至有新消息到达才返回响应
  • 返回后立即发起下一轮请求，形成近实时拉取循环

⚠️ 注意：此模式对网络稳定性要求较高，建议部署在低延迟环境中。

3. 认证与授权机制

本节目标

掌握扫码登录流程与请求认证规范，确保后续API调用合法有效。

登录流程详解

  1. 获取二维码URL ```http GET /ilink/bot/get_bot_qrcode?bot_type=3
    成功响应包含字段`qrcode_img_content`，即二维码图像链接。

2. **生成二维码供扫描**
    使用Python `qrcode` 库将其编码为PNG展示给前端用户：
    ```python
    import qrcode
    img = qrcode.make(qrcode_url)
    img.save("login_qr.png")
  1. 轮询扫码状态 定期调用： ```http GET /ilink/bot/get_qrcode_status?qrcode=
    返回状态包括：

    - `"wait"`：等待扫描
    - `"confirmed"`：已确认，提取凭证
    - `"expired"`：超时失效

4. **提取认证信息**
    状态为`confirmed`时返回关键字段：

    - `bot_token`：Bearer认证凭据
    - `ilink_bot_id`：Bot唯一标识
    - `ilink_user_id`：绑定的微信用户ID
    - `baseurl`：API基地址（如 `https://ilinkai.weixin.qq.com`）

> ⚠️ 注意：**GET请求不得携带`Content-Length`头**，否则会导致服务器等待请求体而超时。

### 请求头规范（所有POST接口）

每次调用需包含以下Header：

| Header | 示例值 | 是否必填 | 说明 |
| --- | --- | --- | --- |
| `Content-Type` | `application/json` | 是 | 固定值 |
| `AuthorizationType` | `ilink_bot_token` | 是 | 固定值 |
| `Authorization` | `Bearer <bot_token>` | 是 | 携带从扫码获取的Token |
| `X-WECHAT-UIN` | `base64(randomUint32_str)` | 是 | 每次请求生成新的随机32位无符号整数字符串，并Base64编码，用于防重放攻击 |
| `Content-Length` | `128` | 是 | 请求体UTF-8编码后的字节长度（字符串形式） |


#### X-WECHAT-UIN生成方法（Python）
```python
import base64
import random

def generate_uin():
    uin_str = str(random.randint(0, 2**32 - 1))
    return base64.b64encode(uin_str.encode()).decode()

安全机制说明

  • 防重放攻击：X-WECHAT-UIN每次请求变化，防止中间人截获并重复提交相同请求。
  • Token有效期：目前未公开具体过期时间，建议持久化存储并在异常时重新扫码。
  • 建议实践：将bot_token加密保存于安全存储中，避免硬编码。

5. 典型业务场景调用流程

本节目标

理解常见交互场景的技术实现路径，避免落入典型陷阱。

场景一：接收用户消息并自动回复文本

流程图解

  1. 调用getupdates拉取消息
  2. 解析item_list获取用户输入
  3. 构造回复消息体
  4. 调用sendmessage发送，必须带回context_token

关键代码逻辑

for msg in response['msgs']:
    reply = {
        "msg": {
            "to_user_id": msg["from_user_id"],
            "client_id": generate_client_id(),
            "message_type": 2,
            "message_state": 2,
            "context_token": msg["context_token"],  # 必须原样带回
            "item_list": [{"type": 1, "text_item": {"text": "收到"}}]
        },
        "base_info": {"channel_version": "1.0.2"}
    }
    send_message(reply)

⚠️ 遗漏context_token将导致消息无法显示在正确对话窗口！

场景二：发送图片消息

四步流程详解

  1. 本地AES加密图像

    • 使用AES-128-ECB模式
    • 密钥：16字节随机值
    • 填充：PKCS7

  2. 调用getuploadurl获取上传参数

    • 提交明文大小、MD5、密文大小、密钥hex字符串

  3. 上传至CDN

    • 目标URL：https://novac2c.cdn.weixin.qq.com/c2c/upload
    • 参数：encrypted_query_param=<upload_param>&filekey=<random_hex_16bytes>
    • Body：加密后的二进制流
    • 成功后从响应头x-encrypted-param获取download_param

  4. 调用sendmessage引用媒体 ```json { "type": 2, "image_item": { "media": { "encrypt_query_param": "download_param", "aes_key": "base64(aeskey.hex())", "encrypt_type": 1 } } }

#### 易错点警示
> 🔥 **aes_key必须先转为十六进制字符串，再对该ASCII字符串进行Base64编码**  
> ❌ 错误做法：`base64.b64encode(aeskey)`  
> ✅ 正确做法：`base64.b64encode(aeskey.hex().encode('ascii'))`

### 场景三：AI生成期间显示“正在输入…”

#### 实现步骤

1. 收到用户消息后，立即调用`getconfig`获取`typing_ticket`
2. 启动后台任务处理AI请求
3. 在等待期间，每隔5秒调用`sendtyping(status=1)`
4. AI生成完成后，调用`sendtyping(status=2)`并发送最终回复

#### 工程建议

- 使用异步任务队列（如Celery）管理长时间任务
- 设置最大等待时间（如60秒），超时后自动取消输入态

## 6. Python完整调用示例

### 本节目标
提供可运行的脚手架代码，帮助开发者快速启动项目。

```python
import requests
import json
import base64
import random
import hashlib
from Crypto.Cipher import AES
import os

class WeChatILinkClient:
    def __init__(self, base_url="https://ilinkai.weixin.qq.com"):
        self.base_url = base_url
        self.session = requests.Session()
        self.bot_token = None
        self.cursor = ""

    def generate_uin(self):
        """生成X-WECHAT-UIN"""
        uin_str = str(random.randint(0, 2**32 - 1))
        return base64.b64encode(uin_str.encode()).decode()

    def set_token(self, token):
        self.bot_token = token

    def _make_headers(self, body):
        length = len(body.encode('utf-8')) if isinstance(body, str) else len(body)
        return {
            'Content-Type': 'application/json',
            'AuthorizationType': 'ilink_bot_token',
            'Authorization': f'Bearer {self.bot_token}',
            'X-WECHAT-UIN': self.generate_uin(),
            'Content-Length': str(length)
        }

    def get_qrcode(self):
        """获取登录二维码"""
        url = f"{self.base_url}/ilink/bot/get_bot_qrcode"
        params = {'bot_type': 3}
        resp = requests.get(url, params=params)
        data = resp.json()
        if data['ret'] == 0:
            return data['qrcode_img_content']
        raise Exception(f"获取二维码失败: {data}")

    def poll_qrcode_status(self, token):
        """轮询扫码状态"""
        url = f"{self.base_url}/ilink/bot/get_qrcode_status"
        params = {'qrcode': token}
        resp = requests.get(url, params=params)
        return resp.json()

    def get_updates(self):
        """长轮询拉取消息"""
        url = f"{self.base_url}/ilink/bot/getupdates"
        payload = {
            "get_updates_buf": self.cursor,
            "base_info": {"channel_version": "1.0.2"}
        }
        headers = self._make_headers(json.dumps(payload))
        try:
            resp = self.session.post(url, headers=headers, json=payload, timeout=36)
            result = resp.json()
            if result.get('ret') == 0:
                self.cursor = result.get('get_updates_buf', '')
                return result['msgs']
            else:
                print(f"拉取消息失败: {result}")
        except requests.Timeout:
            pass  # 忽略超时，继续下一轮
        return []

    def send_message(self, to_user_id, context_token, text):
        """发送文本消息"""
        url = f"{self.base_url}/ilink/bot/sendmessage"
        payload = {
            "msg": {
                "to_user_id": to_user_id,
                "client_id": f"msg_{os.urandom(8).hex()}",
                "message_type": 2,
                "message_state": 2,
                "context_token": context_token,
                "item_list": [{
                    "type": 1,
                    "text_item": {"text": text}
                }]
            },
            "base_info": {"channel_version": "1.0.2"}
        }
        headers = self._make_headers(json.dumps(payload))
        resp = self.session.post(url, headers=headers, json=payload)
        return resp.json()

    def encrypt_image(self, image_data: bytes) -> tuple:
        """AES-128-ECB加密图像"""
        key = os.urandom(16)
        cipher = AES.new(key, AES.MODE_ECB)
        padded_data = self._pad_pkcs7(image_data)
        encrypted = cipher.encrypt(padded_data)
        return encrypted, key

    def _pad_pkcs7(self, data: bytes) -> bytes:
        pad_len = 16 - (len(data) % 16)
        return data + bytes([pad_len] * pad_len)

    def get_upload_url(self, rawsize, rawmd5, filesize, aeskey_hex):
        """获取CDN上传凭证"""
        url = f"{self.base_url}/ilink/bot/getuploadurl"
        payload = {
            "rawsize": rawsize,
            "rawfilemd5": rawmd5,
            "filesize": filesize,
            "aeskey": aeskey_hex,
            "media_type": 2
        }
        headers = self._make_headers(json.dumps(payload))
        resp = self.session.post(url, headers=headers, json=payload)
        return resp.json()

    def upload_to_cdn(self, encrypted_data, upload_param):
        """上传加密图像至CDN"""
        url = "https://novac2c.cdn.weixin.qq.com/c2c/upload"
        filekey = os.urandom(8).hex()
        params = {
            "encrypted_query_param": upload_param,
            "filekey": filekey
        }
        headers = {
            'Content-Type': 'application/octet-stream'
        }
        resp = requests.post(url, params=params, data=encrypted_data, headers=headers)
        download_param = resp.headers.get('x-encrypted-param')
        return download_param

    def send_image(self, to_user_id, context_token, image_path):
        """发送图片全流程封装"""
        with open(image_path, 'rb') as f:
            image_data = f.read()

        # 1. 加密
        encrypted, key = self.encrypt_image(image_data)
        raw_md5 = hashlib.md5(image_data).hexdigest()

        # 2. 获取上传参数
        upload_resp = self.get_upload_url(
            rawsize=len(image_data),
            rawmd5=raw_md5,
            filesize=len(encrypted),
            aeskey_hex=key.hex()
        )
        if upload_resp['ret'] != 0:
            raise Exception(f"获取上传凭证失败: {upload_resp}")

        # 3. 上传CDN
        download_param = self.upload_to_cdn(encrypted, upload_resp['upload_param'])

        # 4. 发送引用消息
        aes_key_b64 = base64.b64encode(key.hex().encode('ascii')).decode()
        payload = {
            "msg": {
                "to_user_id": to_user_id,
                "client_id": f"img_{os.urandom(8).hex()}",
                "message_type": 2,

7. 附录：常见问题与最佳实践

本节目标

汇总高频问题与工程建议，提升系统稳定性与兼容性。

常见问题解答（Q&A）

问题 原因分析 解决方案
GET请求超时 错误添加了Content-Length头 移除该Header，GET请求不应携带
图片发送两次 内存figure与磁盘文件被双重捕获 使用SHA-256哈希去重，计算前1024字节内容指纹
重启后重复处理消息 未持久化get_updates_buf 将游标保存至本地文件（如~/.wechat_cursor.json）
回复消息不显示 遗漏context_token 必须从入站消息中提取并原样带回
解密失败 aes_key编码方式错误 兼容两种格式：
1. base64(raw_bytes)
2. base64(hex_string.encode('ascii'))
无法主动推送消息 协议限制 iLink偏向“会话型回复”，需用户先发起对话

工程最佳实践

  • 游标持久化：将get_updates_buf定期写入磁盘，防止服务重启丢失位置。
  • 幂等键使用：为每条发送消息生成唯一client_id，避免重复提交造成重复发送。
  • 去重机制：对AI生成的内容（如图表）使用内容哈希（如SHA-256前1024字节）判断是否已发送。
  • 日志记录：详细记录请求/响应、Token变更、错误堆栈，便于排查问题。
  • 异常容忍：长轮询允许短暂超时，连续失败3次以上再告警。
